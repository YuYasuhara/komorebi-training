SECTION 03

スプレッドシート関数

📌 Google Sheets / Excel 共通：以下の関数はどちらでも同様に使えます。まずは =SUM, =IF, =VLOOKUP の3つをマスターしましょう。
集計・数値
SUM — 合計
=SUM(範囲)
指定した範囲の数値をすべて足し合わせます。
例: =SUM(B2:B10) → B2からB10の合計
AVERAGE — 平均
=AVERAGE(範囲)
指定した範囲の平均値を返します。
例: =AVERAGE(C2:C20)
COUNT / COUNTA — カウント
=COUNT(範囲)　=COUNTA(範囲)
COUNT は数値セルの個数、COUNTA は空白以外すべてを数えます。
例: =COUNTA(A2:A100) → 入力されているセルの数
MAX / MIN — 最大・最小
=MAX(範囲)　=MIN(範囲)
範囲内の最大値・最小値を返します。
例: =MAX(D2:D50)
条件・論理
IF — 条件分岐
=IF(条件, 真の場合, 偽の場合)
条件が正しければ「真の値」、そうでなければ「偽の値」を表示します。
例: =IF(B2>=60, "合格", "不合格")
COUNTIF — 条件付きカウント
=COUNTIF(範囲, 条件)
条件に一致するセルの数を数えます。
例: =COUNTIF(C2:C50, "東京")
SUMIF — 条件付き合計
=SUMIF(条件範囲, 条件, 合計範囲)
条件に一致する行の数値だけを合計します。
例: =SUMIF(A2:A50, "山田", B2:B50)
検索・参照・表示
VLOOKUP — 縦方向の検索
=VLOOKUP(検索値, 範囲, 列番号, FALSE)
検索値を左端の列で探し、指定した列のデータを返します。最後の FALSE は完全一致。
例: =VLOOKUP(E2, A2:C100, 3, FALSE)
IFERROR — エラー処理
=IFERROR(数式, エラー時の値)
数式がエラーになったときに代替テキストを表示。VLOOKUPとセットで使うことが多い。
例: =IFERROR(VLOOKUP(E2,A:C,3,FALSE), "該当なし")
TEXT — 表示形式の変換
=TEXT(値, "書式")
数値や日付を指定した書式の文字列に変換します。
例: =TEXT(TODAY(), "yyyy年mm月dd日")

SECTION 04

ピボットテーブル

📌 ピボットテーブルとは？ 大量のデータを数クリックで集計・分析できる強力な機能です。関数を書かなくても、ドラッグ＆ドロップで集計表を作れます。
基本の構造
🔢 行フィールド例：都道府県、担当者名
📋 列フィールド例：月、カテゴリ
∑ 値フィールド例：売上金額、件数
🔽 フィルターフィールド例：年度、部署
作成手順（Google スプレッドシート）
1
元データを選択する
集計したいデータ範囲をすべて選択します。ヘッダー行（列名の行）も含めて選択してください。
2
挿入 → ピボットテーブル
メニューから「挿入」→「ピボットテーブル」をクリック。新しいシートまたは既存シートを選択して「作成」。
3
フィールドをドラッグ＆ドロップ
右側のパネルで「行」「列」「値」「フィルター」に各フィールドを追加します。
4
集計方法を選ぶ
値フィールドの集計方法を「合計」「平均」「カウント」「最大/最小」などから選択します。
よく使う活用パターン
やりたいこと行フィールド値フィールド（集計）
担当者別の売上合計担当者名売上金額（合計）
月別の件数推移月案件ID（カウント）
都道府県×カテゴリのクロス集計都道府県売上（合計）＋列にカテゴリ
商品ごとの平均単価商品名単価（平均）
ピボットテーブルの便利ポイント
🔄 データ更新の反映
元データを更新しても、ピボットテーブルは自動更新されません。ピボットテーブル上で右クリック →「更新」を実行するか、Googleスプレッドシートでは自動更新されます。
📐 スライサーでインタラクティブなフィルタ
「データ」→「スライサーを追加」で、ボタンをクリックするだけで絞り込めるUIを追加できます。プレゼンや共有シートで便利です。
📈 ピボットグラフ
ピボットテーブルを選択した状態で「挿入」→「グラフ」を選ぶと、集計結果をそのままグラフ化できます。フィルタを変えるとグラフも連動します。
⚠️ よくある失敗
元データに空白の列・行があると正しく集計されません。また、数値列に文字が混入しているとカウントになってしまうことがあります。データクレンジングを事前に行いましょう。

SECTION 05

応用関数

📌 実務で使う複合関数：以下は調査データやアンケート集計など、複数シートにまたがるデータ分析でよく使われる実践的な関数です。
COUNTIFS — 複数条件でカウント
1条件における指定設問の該当回答者数
COUNTIFS( rawdata_クレンジング済み!M:M, "=指定の値", INDEX(rawdata_クレンジング済み!$O:$X, 0, ROW(A1)), ">=指定の値", INDEX(rawdata_クレンジング済み!$O:$X, 0, ROW(A1)), "<=指定の値" )
M列の条件に一致し、かつO〜X列の中から ROW(A1) で動的に列を切り替えながら数値範囲に該当する回答者数を集計します。

ポイント：INDEX(範囲, 0, 列番号) で列全体を参照。ROW(A1) は下にコピーするたびに 1→2→3… と増えるため、O列→P列→Q列と自動でずれていきます。
▶ 数式を下方向にコピーすると設問ごとの集計が自動で展開されます
2条件における指定設問の該当回答者数
COUNTIFS( rawdata_クレンジング済み!$F:$F, "指定の値", rawdata_クレンジング済み!AC:AC, ">=指定の値", rawdata_クレンジング済み!AC:AC, "<=指定の値", INDEX(rawdata_クレンジング済み!$O:$X, 0, ROW(A1)), ">=指定の値", INDEX(rawdata_クレンジング済み!$O:$X, 0, ROW(A1)), "<=指定の値" )
F列とAC列の2つのフィルター条件を加えたうえで、設問列（O〜X）の回答範囲に該当する人数を集計します。

ポイント：COUNTIFSは「すべての条件を同時に満たす行」を数えます（AND条件）。条件の順番は問いませんが、絞り込みが多い条件を先に書くと処理が速くなる場合があります。
▶ クロス集計・セグメント分析に活用できます
1条件における回答者の人数
COUNTIFS(rawdata_クレンジング済み!$M:$M, "指定の値")
M列が指定の値と一致する行数（＝回答者数）を返します。単一条件の場合は COUNTIF でも同様に使えます。
例: COUNTIFS(rawdata!$M:$M, "男性") → 男性回答者の総数
2条件における回答者の人数
COUNTIFS( rawdata_クレンジング済み!$F:$F, "指定の値", rawdata_クレンジング済み!AC:AC, "指定の値" )
F列とAC列の両方の条件を満たす回答者数を集計します。母数の算出に使います。
例: F列="東京", AC列="20代" → 東京在住20代の人数
LEN / SUM+LEN — 文字数カウント
LEN — 1セルの文字数をカウント
LEN(指定のセル)
指定したセルの文字数（スペース含む）を返します。テキストの長さ確認や、文字数制限チェックに便利です。
例: =LEN(A2) → A2の文字数 / =LEN("こんにちは") → 5
SUM + LEN — 複数セルの合計文字数
SUM(LEN(範囲))
範囲内の全セルの文字数を合計します。配列数式として入力する必要があります。

Googleスプレッドシート：そのまま =SUM(LEN(A2:A100)) と入力
Excel：Ctrl+⇧+Enter で確定（配列数式）
例: =SUM(LEN(B2:B50)) → B列全回答の合計文字数
CHOOSE + INT — 年齢を年代に変換
年齢を年代に変える
CHOOSE(INT(B2/10), "10代", "20代", "30代", "40代", "50代", "60代")
B2の年齢を10で割った整数部分（INT）をインデックスとして、CHOOSE で対応する年代文字列を返します。

仕組み： INT(15/10) = 1 → 1番目の"10代"、 INT(25/10) = 2 → 2番目の"20代"、という対応になります。

注意：10歳未満（INT(9/10)=0）や70歳以上はエラーになります。必要に応じて IFERROR や IF で対処しましょう。
例: B2=34 → INT(34/10)=3 → "30代"
💡 組み合わせ活用例：CHOOSEで年代列を作成してから、COUNTIFSで年代別・設問別の回答者数を集計する、という流れがよく使われます。
